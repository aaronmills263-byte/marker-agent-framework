# Incident: Hook Handler CLI Binaries Missing

## What was broken

The shell scripts generated by `marker-hooks install` invoked runtime CLIs (`pretooluse-cli.js`, `posttooluse-cli.js`) that did not exist as built artefacts. The generated scripts used `npx tsx` to run non-existent source files, causing them to fail silently. As a result, hooks were installed but never actually enforced — all tool calls proceeded unblocked regardless of the configured rules.

## When discovered

2026-04-20

## Impact

Any consuming repo that ran `marker-hooks install` before this fix had **non-functional hooks**. The pre-tool-use checks (deny patterns, protected paths, kill switch enforcement) were not applied. Post-tool-use audit entries were not written. The hooks appeared installed (scripts existed, settings.json referenced them) but provided zero enforcement.

## Root cause

1. The `main()` functions existed in `pretooluse.ts` and `posttooluse.ts` but were never invoked from a standalone entry point — no CLI wrapper files existed.
2. The generated shell scripts referenced paths that didn't correspond to any compiled output.
3. The scripts used `npx tsx` (requiring tsx at runtime) instead of `node` with the compiled JS.
4. No `bin` entries existed in `package.json` for the handler CLIs.

## Fix applied

- Created `src/handlers/pretooluse-cli.ts` and `posttooluse-cli.ts` — thin wrappers that import and call `main()`.
- Added `bin` entries in `package.json` for `marker-pretooluse` and `marker-posttooluse`.
- Updated `generate.ts` to emit shell scripts that invoke `node` with the absolute path to the compiled handler JS (resolved via `node_modules/@marker/hooks/dist/handlers/`).
- Added integration test that installs hooks in a temp directory, pipes JSON payloads to the generated scripts, and verifies correct exit codes.
- Bumped version to `0.2.0`.

## Remediation

Consuming repos must re-run `marker-hooks install` after updating `@marker/hooks` to `>=0.2.0`. The old shell scripts reference non-existent paths and must be regenerated.

## Lesson

Integration testing must invoke the full installed-and-running path, not just unit-test the handler logic in isolation. The unit tests for `handlePreToolUse()` all passed while the actual hook enforcement was completely broken because no test ever executed the generated shell script end-to-end.
